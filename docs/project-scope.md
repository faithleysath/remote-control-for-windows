# 项目范围

Remote Control for Windows 面向明确授权场景下的 Windows 远控。本文统一定义产品模式、目标用户、当前基线能力、近期优先级、后续方向和明确不做的事，不重复协议、发布或实现细节。

## 产品模式

- 临时协助模式：当前已实现。host 提供运行期 TOTP，控制端按次建立会话，关闭 host 即结束可连接状态。
- 长期配对模式：规划中。host 可在用户显式启用后保持长期可连接，控制端使用固定密码、密钥对或类似长期凭据重新建立 session，但 session token 本身仍保持短期。
- 两种模式共享同一组底线：授权明确、状态可见、操作可审计、凭据可撤销，不做静默控制或权限绕过。

## 核心用户

- 客户或测试人员：运行当前 `rcw-host.exe`，分享可见的机器 ID 和 TOTP；后续若启用长期配对，也应能在可见界面中查看当前可连接状态、管理配对并禁用常驻模式。
- 研发或运维人员：使用 `rcwctl` 打开会话，执行诊断、传输文件、截图和基础桌面输入；后续可复用长期配对凭据重新连接同一 host。
- 自动化 agent：通过 `rcwctl` 完成有边界的支持任务，默认应使用 `--json`。
- 服务端管理员：部署 `rcw-server`，管理控制端 token，并查看中继侧日志。

## 当前已实现的工作流

- 被控端通过出站 WebSocket 登记在线。
- 使用控制端 token + 机器 ID/TOTP 创建会话。
- 控制端本地 session 文件支持短生命周期 `rcwctl` 多次调用复用。
- 远程命令执行，包含 stdout、stderr、退出码、超时、清理，以及 server-owned 后台 exec 查询/取消。
- 上传和下载，包含分块传输和 SHA-256 校验。
- 从 Windows 交互桌面截图。
- 枚举可见窗口。
- 使用屏幕绝对坐标执行鼠标移动、点击和滚轮。
- 输入文本和常见按键/快捷键。
- 被控端自动更新安全的剪贴板连接信息。
- 被控端运行期间临时阻止系统睡眠和显示器熄屏。
- host、controller、server 三端审计日志通过 request ID 对齐。

## 当前基线暂不提供

- 常驻服务或系统重启后的自动恢复可连接状态。
- 固定密码、密钥对或类似长期凭据的创建、轮换和撤销。
- 开机自启、托盘常驻或安装包内的常驻配置流。
- 多租户 SaaS 管理、计费或组织管理。
- v1 中的中央审计数据库。
- 当前基线中的 P2P 穿透。

## 近期优先级

- 补回精简后的测试文档，并把剩余验证缺口收口到单一真相源。
- 闭合剩余的标准用户交互桌面验证缺口。
- 围绕现有手工 E2E 清单增加可重复的 Windows VM smoke 自动化。
- 改进发布打包和校验和生成。
- 增加 Linux workspace 检查的 CI 覆盖。
- 为未来协议消息变更增加兼容性测试。
- 加强审计脱敏测试。
- 改进常见 Windows 桌面状态的操作者错误提示，例如锁屏、UAC 安全桌面和非交互 session 0。

## 后续方向

- 保留当前临时协助 / TOTP 模式，避免为了常驻化破坏现有授权链路。
- 增加显式启用的长期配对模式，支持固定密码、密钥对或类似长期凭据。
- 增加长期凭据的创建、轮换、撤销和禁用流程，并保持凭据不进入剪贴板、普通日志和审计摘要。
- 让 host 在 GUI、托盘或安装体验中明确展示当前是否处于长期可连接状态。
- 支持 host 重启或系统重启后的可恢复连接，同时继续保持 session token 短期化和操作可审计。
- 一条命令收集诊断包。
- 更好的多显示器选择和坐标报告。
- 窗口相对坐标辅助。
- 在保持当前安全模型的前提下增加 OCR 或 UI 元素发现辅助。
- 控制端操作 transcript，便于会话后复盘。
- 可选的客户侧命令类别允许/拒绝策略。
- 分发 `rcw-host.exe` 的下载或安装入口。
- 通过同一中继模型提供端口转发，并保留明确审计和客户可见状态。
- 为需要集中留存的部署增加服务端持久审计存储。

## 明确不做

- 静默控制或隐藏运行。
- 不可见的后台控制。
- 未经用户显式启用、不可撤销的常驻化或开机自启。
- UAC 绕过或自动权限提升。
- 内核驱动、进程注入或键盘记录。
- 实时视频流或后台录屏。

更细的安全边界见 [security.md](security.md)，实现与 crate 分工见 [architecture.md](architecture.md)。
