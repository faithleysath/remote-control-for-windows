# 产品需求

## 背景

研发经常需要通过向日葵远控帮助客户处理产品问题。客户环境通常是 Windows，传统远控只能人工操作，Codex agent 无法直接介入。该产品要提供一套临时、可见、可控的远控链路，让研发可以把诊断和修复动作交给 Codex 通过 CLI 执行。

## 目标

- 让客户无需安装复杂软件，只需双击运行一个 Windows 被控端。
- 让 Codex 可以通过控制端 CLI 执行远程诊断和操作。
- 让远控连接适应客户内网环境，不要求客户开放入站端口。
- 让研发能采集必要诊断信息，包括命令输出、日志文件、截图等。
- 让所有远控操作在被控端、控制端和服务端三端都有日志留档，并在被控端控制台实时可见。
- 保持客户可见、可关闭、非持久化，避免远控工具变成常驻后门。

## 用户角色

- 客户现场用户：运行 `rcw-host.exe`，把机器 ID 和当前 TOTP 提供给研发，需要时关闭窗口终止控制。
- 研发/运维人员：通过 `rcwctl` 建立会话，执行诊断、文件传输、截图和桌面操作。
- Codex agent：通过 `rcwctl` 子命令自动完成排障步骤。
- 服务端管理员：部署 `rcw-server`，管理控制端 token 和服务器地址。

## 核心场景

### 临时诊断

客户双击打开被控端，把机器 ID 和 TOTP 发给研发。研发建立会话后执行系统命令、读取日志、下载诊断文件，并在完成后关闭会话。

### 远程修复

研发上传补丁、脚本或工具，通过命令行执行修复动作，并截图确认界面状态。

### 图形界面协助

当问题只能在 GUI 中复现或处理时，研发通过截图、鼠标点击、键盘输入和窗口列表完成操作。Codex 可以逐步截图、定位窗口、点击控件、输入文字。

### 客户主动终止

客户关闭被控端控制台窗口后，服务器立即注销该主机和相关会话，控制端后续命令失败。

## 功能需求

### 被控端

- 启动后无需 token 即可连接服务器并登记在线。
- 根据 Windows 机器信息生成稳定机器 ID。
- 每次运行生成随机 TOTP 种子。
- TOTP 有效周期默认 120 秒，并支持配置覆盖。
- 控制台持续显示机器 ID、当前 TOTP、服务器地址、连接状态和当前会话状态。
- 启动后自动把连接信息复制到 Windows 剪贴板；每次 TOTP 刷新后同步更新剪贴板。
- 控制台显示剪贴板复制状态，失败时提示客户手动复制 ID/TOTP。
- 运行期间阻止系统休眠和显示器熄屏，避免远控会话中断。
- 控制台实时追加显示所有远控操作摘要，包括命令类型、开始时间、结束状态和关键目标路径或窗口信息。
- 本地写入被控端审计日志。
- 启动时检测当前是否以管理员权限运行；如果是管理员权限，控制台必须使用醒目颜色高亮提示。
- 认证成功后接受同一会话内的远控命令。
- 支持关闭窗口直接终止远控。
- 不写入开机启动项，不注册服务，不隐藏窗口。
- 不做自动 UAC 提权；需要管理员权限时，由客户自行右键以管理员身份运行。

### 控制端 CLI

- 支持通过机器 ID、TOTP 和控制端 token 建立会话。
- 支持把会话状态保存到本地文件，后续命令自动复用。
- 支持执行远程命令并返回 stdout、stderr、退出码。
- 支持上传、下载文件。
- 支持截取当前屏幕。
- 支持列出窗口和基础窗口信息。
- 支持鼠标移动、点击、滚轮。
- 支持键盘输入文本和快捷键。
- 支持关闭当前会话。
- 本地写入控制端审计日志，记录发起的操作、目标机器 ID、请求 ID、时间和结果。

### 服务端

- 支持被控端和控制端通过 WebSocket 主动连接。
- 负责主机登记、控制端鉴权、会话创建、消息中继；被控端上线不要求 token。
- 首版只保存内存状态，服务器重启后所有主机和会话失效。
- 提供健康检查接口。
- 提供结构化日志，记录主机连接、会话创建、命令类型和断开原因。
- 服务端写入中继审计日志，记录认证、会话、命令中继和断开事件。

## 非功能需求

- 网络模型优先适配内网客户环境，所有端都只需主动出站连接。
- 控制端 CLI 必须适合脚本化和 Codex 调用，避免必须进入交互式 TUI。
- 协议使用结构化消息，避免依赖脆弱的字符串解析。
- 文件传输必须做分块和哈希校验。
- 命令执行必须支持超时和取消。
- 首版不追求低延迟视频流，截图按需获取即可。
- 三端审计日志必须使用结构化格式，便于后续采集和排查。

## 成功标准

- 客户可在 Windows 上双击启动被控端并看到 ID/TOTP。
- 研发可在 Linux 上通过 `rcwctl` 建立会话。
- Codex 可用 CLI 完成命令执行、文件传输、截图和基本鼠标键盘操作。
- 关闭被控端窗口后，控制端会话立即不可用。
- 服务端地址可在打包时嵌入，也可通过环境变量覆盖。
- TOTP 周期可配置，默认 120 秒。
- 被控端启动后能自动复制可转发给研发的连接信息到剪贴板，且不包含控制端 token。
- 被控端运行期间系统不会自动休眠或熄屏；被控端退出后不保留该电源请求。
- 同一次会话中的命令执行、文件传输、截图、鼠标键盘操作能在三端日志中按 request ID 对齐。
