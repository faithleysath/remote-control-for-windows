# 安全策略

Remote Control for Windows 面向明确授权的 Windows 远控场景。当前主链路已经实现临时支持会话，路线图还会增加常驻可连接和长期配对能力；无论哪种模式，涉及安全问题的报告都应先私下提交给维护者，在维护者确认并准备修复或缓解措施前，不应公开披露。

## 支持版本

当前 `main` 分支是活跃开发和维护线。目前还没有单独维护稳定发布分支。

## 报告漏洞

在公开安全联系人配置完成前，请通过维护者现有的私有项目支持渠道报告漏洞。涉及鉴权绕过、token 暴露、静默行为、未经显式启用的常驻化或自启动、权限提升、命令执行、文件传输或审计脱敏的问题，不要直接提交公开 issue。

报告中请包含：

- 受影响的 commit 或 release。
- 复现步骤。
- 预期行为和实际行为。
- 是否可能暴露控制端 token、session token、TOTP seed、原始机器标识或客户数据。

## 安全边界

项目不接受会引入静默控制、隐藏后台控制、未经显式启用和可撤销机制的常驻化、自动 UAC 提权、UAC 绕过、键盘记录、驱动安装或进程注入的功能。

完整产品安全模型见 [docs/security.md](docs/security.md)。
